# SafeAgent-CS v0.4A Execution Fact Source

## 1. 为什么先补执行事实源

v0.3 已经具备 checkpoint / resume readiness，但还不能安全开启真实恢复。原因是工具调用和 pending action 都是副作用边界：如果恢复时无法判断某个工具是否已经执行、某个 pending action 是否已经创建，就可能重复调用工具或重复创建待确认动作。

v0.4A 因此先补事实源，而不是直接启用 resume。

## 2. tool_call_id 与 idempotency_key 的区别

`tool_call_id` 表示一次物理工具调用 ID。每次真正进入 ToolGateway 执行工具时都会生成新的 `tool_call_id`。

`idempotency_key` 表示一次逻辑业务动作的稳定幂等键。同一用户、同一租户、同一工具、同一 action fingerprint，在 retry 或未来 resume 场景中应保持相同。

## 3. action_fingerprint 设计

`action_fingerprint` 基于 ActionPlan 或 ToolGateway 调用上下文生成。

设计原则：

- 使用 canonical JSON。
- `sort_keys=True`。
- 生成 hash，不保存原始参数。
- 剔除 `raw_message`、`message`、`run_id`、`source_run_id`、`request_id` 等易变或高风险字段。
- 先脱敏再 hash。

当前格式：

```text
af_<sha256>
```

## 4. ToolGateway 如何写入工具调用事实

ToolGateway 每次调用工具前生成：

```text
tool_call_id
idempotency_key
action_fingerprint
attempt_no
```

并写入 `tool_call_logs`。

本阶段只记录事实，不改变执行语义：

- 不因为相同 `idempotency_key` 跳过工具。
- 不复用历史 ToolResult。
- 不启用真实 resume。

## 5. retry 时的关系

FailureHandler retry 仍必须通过 ToolGateway。

retry 行为：

- `tool_call_id`：新的物理调用 ID。
- `idempotency_key`：同一逻辑动作保持稳定。
- `action_fingerprint`：同一逻辑动作保持稳定。
- `attempt_no`：递增。

## 6. pending_action_events 的作用

`pending_actions` 是当前状态表，只能说明一个 pending action 现在是什么状态。

`pending_action_events` 是事件事实表，用于说明状态如何变化：

- `CREATED`
- `CONFIRMED`
- `CANCELLED`
- `EXPIRED`
- `EXECUTED`
- `POLICY_RECHECK_FAILED`（预留）

事件记录包含：

```text
event_id
pending_action_id
run_id
parent_run_id
session_id
user_id
tenant_id
event_type
old_status
new_status
reason
metadata_json
created_at
```

## 7. 为什么本阶段不启用真实 resume

真实 resume 需要同时具备：

- 工具调用事实源。
- 工具幂等策略。
- pending action 事件历史。
- checkpoint snapshot schema 迁移策略。
- run 生命周期恢复策略。
- side-effect node 安全恢复策略。

v0.4A 只完成前两类事实源准备，不直接打开 `/api/resume`。

## 8. 后续如何支持真实 checkpoint/resume

建议顺序：

1. 固化 `tool_call_id` / `idempotency_key` 查询协议。
2. 补 pending_action_events 在 confirm API 和 confirm workflow 中的审计查询。
3. 将 checkpoint snapshot 与 tool/pending facts 建立关联。
4. 对 side-effect node 做真实恢复策略：
   - 工具已成功：跳过重复工具调用。
   - 工具失败且 retryable：通过 ToolGateway 重试。
   - pending action 已创建：恢复到等待确认状态。
5. 最后再接真实 LangGraph checkpointer。
