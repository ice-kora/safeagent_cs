## 3. manual / workflow / langgraph 对照

| 路径 | 默认启用 | 作用 |
| --- | --- | --- |
| manual | 是 | 当前生产安全主链路 |
| workflow style | 否 | 轻量 Workflow 编排验证 |
| real langgraph | 否 | 真实 LangGraph chat workflow engine |

验收原则：

- manual 测试必须稳定通过。
- workflow 不能绕过 Validator、PolicyService、ToolGateway。
- LangGraph 节点语义必须与现有 service 边界一致。

## 4. Safety Regression

Safety Regression 验证：

- DENY 不进入 ToolGateway。
- CONFIRM_REQUIRED 创建 pending action 且不调用工具。
- HUMAN_REQUIRED 不自动执行工具。
- ALLOW 必须经过 ToolGateway。
- intentional difference 不能隐藏真实失败。

## 5. RAG Evaluation

RAG 测试验证：

- policy query 能返回 answer。
- sources 不为空。
- citation quality 可评估。
- RAG 不返回敏感订单信息。
- RAG 不参与权限裁决。

## 6. Checkpoint / Resume Readiness Tests

相关测试验证：

- snapshot 可 JSON 序列化。
- snapshot 脱敏。
- schema version 固定。
- side-effect node 默认不允许真实 resume。
- dry-run 不调用 ToolGateway。
- dry-run 不创建 pending action。
- tool resume 需要幂等元数据。
- pending resume 需要 pending_action_id。
