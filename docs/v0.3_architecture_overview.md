# SafeAgent-CS v0.3 Architecture Overview

## 1. 项目定位

SafeAgent-CS 是一个企业级受控客服 Agent 执行平台。它的核心价值不是“让模型直接办事”，而是把意图识别、计划生成、RAG、工具调用和人工确认纳入确定性安全边界。

一句话：LLM、Planner、RAG 只能提出候选；Validator、PolicyService、ToolGateway、PendingActionService、FailureHandler 才决定是否执行、确认、转人工、拒绝或失败收口。

## 2. 分层架构

```text
User
  -> API Layer
  -> Planning Layer
  -> Guard / Validation Layer
  -> Policy Kernel
  -> Execution Gateway
  -> Workflow / Trace / Evaluation
```

分层说明：

- API Layer：`/api/chat` 和 `/api/confirm`。
- Planning Layer：RuleBasedIntentClassifier、RuleBasedActionPlanner、后续 LLM Planner。
- Guard / Validation Layer：LLMOutputGuard、ActionPlanValidator、LLMResponseGuard。
- Policy Kernel：PolicyService、RepositoryService。
- Execution Gateway：ToolGateway、Mock Tools、FailureHandler。
- Workflow Layer：manual、LangGraph-style、真实 LangGraph chat workflow。
- Readiness Layer：checkpoint snapshot、resume protocol、resume matrix。

## 3. 主链路

```text
User
 -> /api/chat
 -> IntentClassifier
 -> ActionPlanner
 -> LLMOutputGuard
 -> ActionPlanValidator
 -> PolicyService
 -> route
    -> ALLOW -> ToolGateway -> FailureHandler
    -> CONFIRM_REQUIRED -> PendingAction
    -> HUMAN_REQUIRED -> HumanRequired
    -> DENY -> Deny
 -> ResponseGeneration
 -> LLMResponseGuard
 -> Trace/Logging
```

`/api/confirm` 是独立恢复入口，不重新做 NLU（自然语言理解）或 Planner。它读取已保存的 `action_plan_json`，重新经过 PolicyService 复核，再通过 ToolGateway 执行。

## 4. 安全内核

安全内核由以下组件组成：

- ActionPlanValidator：判断计划是否像一个合法计划。
- PolicyService：判断当前用户是否能执行该计划。
- ToolGateway：唯一工具入口，负责白名单、路由、日志和脱敏。
- PendingActionService：保存二次确认动作。
- FailureHandler：负责失败重试、降级和失败日志。
- TraceService / LoggingService：记录链路和脱敏日志。

关键边界：

- Validator 不判断真实权限。
- PolicyService 不调用业务工具。
- ToolGateway 不判断权限。
- FailureHandler 不绕过 ToolGateway。
- RAG 不参与权限裁决。

## 5. LangGraph 编排层

v0.3 支持真实 LangGraph chat workflow engine。默认仍为 manual / style，不默认启用真实 LangGraph。

LangGraph 负责：

- 节点编排。
- 条件路由。
- 状态传递。
- 为 checkpoint / resume 打基础。

LangGraph 不负责：

- 权限判断。
- 风险裁决。
- 工具幂等。
- pending action 事实管理。
- LLM 安全裁决。

这些能力继续复用现有 service。

## 6. RAG 工具层

RAG KnowledgeTool 只服务公开政策查询：

- 查询七天无理由、退款政策、发票、售后规则等知识。
- 返回 answer 和 sources。
- 支持本地 chunk、retrieval、rerank、citation quality eval。
- 不读取完整订单。
- 不参与权限判断。
- 不替代 PolicyService。

## 7. Checkpoint / Resume Readiness

当前完成 readiness：

- JSON-safe checkpoint snapshot。
- `checkpoint.snapshot.v1` schema version。
- checkpoint / resume protocol。
- InMemoryCheckpointStore。
- resume dry-run。
- tool resume readiness。
- pending action resume readiness。
- resume matrix。

当前未启用：

- 真实 LangGraph checkpointer。
- MemorySaver / SQLiteSaver。
- `/api/resume`。
- 从 checkpoint 真实恢复执行。

## 8. 当前非目标

v0.3 不做：

- 真实 LLM 接入。
- 外部向量数据库。
- MCP 工具接入。
- 前端控制台。
- 真实支付、退款、订单系统接入。
- 完整客服后台权限模型。
- 真实 checkpoint / resume 执行。
