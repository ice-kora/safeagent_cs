# SafeAgent-CS v0.3 Roadmap

## 1. 已完成

- Controlled Agent execution kernel。
- `/api/chat` 主链路。
- `/api/confirm` 二次确认恢复入口。
- Workflow mode。
- Confirm workflow。
- Safety regression。
- RAG KnowledgeTool。
- RAG evaluation。
- Real LangGraph chat workflow。
- Checkpoint snapshot readiness。
- Checkpoint / resume protocol。
- In-memory checkpoint store dry-run。
- Tool resume readiness。
- Pending action resume readiness。

## 2. 下一阶段可选项

### 2.1 Real LLM Provider Adapter

- 价值：展示真实模型理解复杂表达的能力。
- 风险：非法 JSON、幻觉、Prompt Injection、输出越权计划。
- 前置条件：LLMOutputGuard、ActionPlanValidator、PolicyService、LLMResponseGuard 必须保持强制链路。
- 是否适合现在做：适合做 P0.5，但必须先保持 Rule Mode 稳定。

### 2.2 Real Vector Retrieval

- 价值：提升知识库规模和召回质量。
- 风险：检索结果污染回复、citation 质量下降、把 RAG 误用成权限系统。
- 前置条件：文档清洗、chunk 规范、source 质量评估、RAG eval。
- 是否适合现在做：可以做，但仍限定在 policy query。

### 2.3 Real Checkpoint With Checkpointer

- 价值：让 LangGraph 支持更完整的中断和恢复。
- 风险：重复工具调用、重复 pending action、恢复状态不一致。
- 前置条件：tool_call_id、idempotency_key、pending_action_events、resume matrix。
- 是否适合现在做：不适合立即打开真实恢复，适合继续补事实源。

### 2.4 Tool Idempotency Persistence

- 价值：为真实 resume、重试和故障恢复提供幂等依据。
- 风险：幂等键设计错误会导致漏执行或重复执行。
- 前置条件：明确 action + target + user + run 的幂等模型。
- 是否适合现在做：适合优先做。

### 2.5 Pending Action Event Sourcing

- 价值：补齐 pending action 状态历史，支持审计和恢复。
- 风险：事件与状态表不一致。
- 前置条件：定义 event_type、old_status、new_status、run_id、reason。
- 是否适合现在做：适合优先做。

### 2.6 MCP Tool Adapter

- 价值：接入外部工具生态。
- 风险：工具能力边界扩大，安全审计复杂度上升。
- 前置条件：MCP adapter 必须挂在 ToolGateway 后面。
- 是否适合现在做：不适合早于 ToolGateway 生产级幂等与权限边界。

### 2.7 Frontend Console

- 价值：展示 runs、traces、policy logs、tool logs、failure logs。
- 风险：展示敏感数据、权限隔离不足。
- 前置条件：日志脱敏、查询权限、分页和审计。
- 是否适合现在做：适合在后端日志查询接口稳定后做。

### 2.8 AuthN/AuthZ Hardening

- 价值：生产多租户与多角色安全基础。
- 风险：身份模型复杂，错误授权影响严重。
- 前置条件：actor_id、actor_role、subject_customer_user_id、merchant_tenant_id 等字段模型统一。
- 是否适合现在做：上生产前必须做。

### 2.9 Multi-tenant Isolation

- 价值：防止跨商家、跨客户访问数据。
- 风险：tenant_id 传播遗漏。
- 前置条件：RepositoryService、PolicyService、日志与查询接口统一 tenant 约束。
- 是否适合现在做：企业落地前必须强化。
